# Why the JouleBridge runtime is written in Rust

The runtime sits close to physical equipment. That changes the bar.

We care about:

- deterministic behavior
- low runtime overhead
- explicit failure modes

Rust is a good fit for that combination.

This is not about language fashion. It is about keeping site behavior predictable while the proof path remains trustworthy.
