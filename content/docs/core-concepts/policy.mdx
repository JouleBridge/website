# Policy

Policy is not just a reporting layer. It sits in the execution path.

## What policy does

- constrains local action
- defines default behavior
- rejects unsafe or out-of-scope requests
- keeps site behavior legible

## Why order matters

If policy runs too late, you only learn after the site has already done the wrong thing.

JouleBridge treats policy as part of local coordination, not just post-facto analysis.

## Operational rule

The safest default is to deny by default when the site is in an uncertain state.
