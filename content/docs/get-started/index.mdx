# Get Started

These guides focus on the first serious deployment motion for JouleBridge: bring the runtime onto a real site, validate the local control path, and confirm that the proof chain is useful after the site acts.

## What you are starting

You are not starting a generic SaaS product.

You are starting a local-first control and proof stack:

- Asset Agent near the equipment
- Site Router for local coordination
- Portfolio Orchestrator for policy and proof ingest when needed

## Recommended order

1. Read the [Beginner Guide](/docs/get-started/beginner-guide).
2. Follow the [Quick Start](/docs/get-started/quickstart).
3. Review the [Architecture](/docs/core-concepts/architecture) and [Security](/docs/operations/security) pages before pilot deployment.

## What success looks like

A successful first setup means:

- the runtime is deployed on the intended site hardware path
- at least one real asset interface is connected
- the local runtime can seal a signed proof record
- the deployment can be repeated without guesswork
