# Quick Start

This quick start is about validating the runtime path, not finishing a production rollout.

## 1. Prepare the environment

- Linux host or partner gateway
- network access to at least one target asset or simulator
- deployment config for the intended site profile

## 2. Provision the runtime

Use the provisioning flow for the target site identity and policy bundle.

The exact command surface may change as the pilot tooling hardens, but the intent stays the same:

- enroll the site
- pull the active config and policy
- initialize local runtime state

## 3. Start the local services

Bring up the site-facing services needed for the deployment:

- Asset Agent
- Site Router
- any local proof export or sync companion used in the current pilot path

## 4. Validate one real signal path

Confirm that one meaningful asset path works end to end:

- read site state
- issue or simulate one safe local action
- confirm acknowledgement or measured result
- export the resulting proof record

## 5. Confirm repeatability

The quick start is complete only when another operator can repeat the same path without tribal knowledge.

After that, move to:

- [Architecture](/docs/core-concepts/architecture)
- [Security](/docs/operations/security)
- [Deployment](/docs/operations/deployment)
